# Product Overview

_Evidence-based summary. All statements are verifiable from the repository contents._

---

## What is it

A **Financial Metrics Dashboard** вҖ” full-stack SPA that visualizes 4 financial KPIs and 2 monthly line charts from synthetic data.

**Evidence:** `frontend/src/App.tsx` renders 4 `KPICard` components (Total Income, Total Outcome, Profit, Profit Margin) via `KPIRow`, plus `IncomeOutcomeChart` and `ProfitPercentChart`. Data comes from `GET /api/metrics` which returns 360 mock movements (30/month Г— 12 months) generated by `backend/app/routes.py` with fixed seed 42.

## Limitations (verifiable)

| Limitation | Evidence |
|---|---|
| No database вҖ” all data is ephemeral mock | No `database` service in `docker-compose.yml`, no ORM in `requirements.txt` |
| No authentication вҖ” all endpoints public | No auth middleware in `main.py` |
| CORS wide open | `main.py` L9: `allow_origins=["*"]` |
| Single view вҖ” no routing | `package.json` has no `react-router-dom` dependency |
| No state management library | Only React built-in hooks (`useState`, `useEffect`) in `App.tsx` |
| `<title>` placeholder | `index.html` L7: `<title>frontend</title>` |
| `.dark` class hardcoded | `App.tsx` L44: `className="dark min-h-screen..."` |
